Images

<img> with srcset & sizes is better when you just want different sizes of the same image for performance reasons, <picture> is best when you want entirely different images/aspect ratios depending on viewport.

<img> with srcset & sizes

Used when you want higher performance by loading an image appropriately sized for the device it'll be displayed on. If you want differently sized source images but your displayed image size will be the same regardless, you can omit sizes.

srcset
Gives the browser a selection of image files to choose from. Consists of an image filename, space, and the intrinsic size of the image found at the filename.
sizes 
            sizes="(max-width: 600px) 480px,
            800px"
Defines a set of media conditions and lists which image from srcset would be best for each. Comprised of a media condition, a space, and the width of the slot the image will fill if the media condition is true.

<picture> 

        <picture>
            <source media="(max-width: 799px)" srcset="elva-480w-close-portrait.jpg" >
            <source media="(min-width: 800px)" srcset="elva-800w.jpg" >
            <img src="elva-800w.jpg" alt="Chris standing up holding his daughter Elva" >
          </picture>
  • Wraps a series of source tags and a final image tag
  • <source> includes a media attribute which sets the condition under which the image linked in the following srcset attribute will be loaded.
  • The final <img> tag with (src and alt) must be included as a fallback, or nothing will be displayed by the picture tag
