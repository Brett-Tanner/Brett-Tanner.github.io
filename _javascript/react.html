Components

Components are self contained functions which return chunks of HTML. They can be passed props, and render other components inside them. The App component is the root component of the app which all others are rendered inside. The component function should be capitalized.

Returning null, undefined and false will cause nothing to be rendered, while other falsy values like 0 or empty strings will be rendered.

Hooks

Always start with 'use'. Can only be called from the top level of a functional component, and not within a loop or conditional.

STATE SHOULD ALWAYS BE IMMUTABLE

If you're updating an object or array in state, separate it out with the spread operator and overwrite the stuff you want to change, then return the new object rather than editing the existing one.

If you call setState(state + 1) twice in a row, it'll only increment by 1 since the stored state is used for both (and replaced, not updated). If you want to update it, use an updater callback like setPerson((prevPerson) => ({ ...prevPerson, age: prevPerson.age + 1 }));

Keys

Keys are used to uniquely identify sibling elements of the same type during re-renders. The UUID package can be used to create unique ids if you're not working with a DB that provides them. Make sure you set the keys outside the component, if you set them in the JSX itself it'll be re-rendered each time which defeats the point.

Indexes as keys are usually a bad idea, but in some situations can lead to better performance without the drawbacks. For example a paginated list where the components don't have state, it's faster to just use the index as a key so the components can just have their contents replaced, rather than being unmounted and new components mounted.

Props

Props can be destructured by using an object with the names of the props you want as the param of the component function (like Button({ text, classes })).

Default props can be set by setting the defaultProps property of the component. It's also possible to combine destructuring and default props like Button({ text = "Click Me!", classes = "text-lg" }).

JSX

JSX is a filetype that lets you write HTML-like markup in a JS file. Use curly braces ({}) to embed JS expressions or variables, but only as text directly inside a JSX tag or as attributes immediately following an '='.

JSX Rules

Return a single root element
If you want to return multiple elements, wrap them in a div or a React Fragment
Close all tags
Tags that're self-closing in HTML aren't in JSX
camelCase most things
Because JSX turns into JS, you can't use stuff like class or '-' which you usually would in HTML. So class becomes className and other stuff is camel cased. aria and data attributes are written with dashes as in HTML for historic reasons.
