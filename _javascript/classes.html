Classes are a type of special function which serve as templates for creating objects. They can have static (class) or instance, as well as public/private methods. Methods are defined on the prototype of each instance and are shared by all instances. Fields can be declared without let or const as they're more like properties on an object than variables.

Constructor

A special method for instantiating an object created with a class. Can only be one constructor in a class. You can use super to call the constructor of a parent class.

Private Fields

Private fields are declared prefixed with a hash (#). Like in Ruby, they can only be read or written within the class body. They can only be declared up-front in a field declaration, and can't be created later by assigning to them after the object is instantiated.

Inheritance

The extends keyword is used to create a class as a child of another constructor, like class Dog extends Animal. If there's a constructor in the child class it needs to call super() and pass the params required by the parent constructor before using this. super() can also be used to call corresponding methods of the parent class like super.method()

Getters and Setters

These are class methods which, when called from outside the class, just look like accessing a property which shares their name rather than calling a function. For example, the seasonal site uses a setter for both parents and children to combine the two name fields into a full name which is persisted in the DB. They can be defined with get(){} or set(){}.

Static

static methods/fields are the equivalent of a class method/variable in Ruby; they're defined on the class itself rather than an instance.
