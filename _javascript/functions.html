Function Expressions

Function expressions are a function declaration wrapped in brackets (and with brackets at the end if you want it to execute immediately) so it doesn't pollute the global scope. I've most often seen them as anonymous callbacks in other functions like addEventListener, but it's considered best practice to give them a descriptive name as it aids with debugging/readability and allows the function to call itself recursively.
